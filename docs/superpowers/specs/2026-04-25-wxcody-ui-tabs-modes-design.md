## 背景

对 `WXCody/pages/console/console` 做 UI 改造：

- 首页底部 Tab 只保留「模式」与「设置」
- 「模式」Tab 内每个模式项右侧新增「设置」按钮
  - 点击模式按钮：仍执行切换模式（现有 `onSetMode`）
  - 点击设置按钮：进入该模式对应的设置页面
  - 若该模式无设置页面：提示“功能还未开放”

本改造优先复用现有页面结构与 BLE 逻辑，尽量不改业务协议与数据模型。

## 当前结构（已确认）

- 页面：`pages/console/console.*`
- 底部 Tab 按钮位于 `console.wxml` 的 `tabsBottom` 区域
  - 当前 Tab：0=模式、1=图库、2=笔记、3=手绘、4=设置
- 模式切换：
  - 模式按钮在 `console.wxml` 的 `modeGrid` 中
  - 点击触发 `console.js` 的 `onSetMode`，向设备发送 `{ cmd: "set_mode", mode }`

## 目标交互

### 1) 底部 Tab（首页 Tab）

- 仅保留 2 个按钮：
  - 「模式」（tab=0）
  - 「设置」（tab=4）
- 其他 Tab（1=图库、2=笔记、3=手绘）不再出现在底部，但其内容区域继续保留（作为“模式设置页”承载）。

### 2) 模式 Tab：模式项 + 设置按钮

模式列表呈现为“每行一个模式”，每行包含：

- **左侧主按钮**：模式名称（带 emoji），点击执行 `onSetMode`（切换模式）
- **右侧设置按钮**：小按钮（例如“设置”或图标），点击进入该模式设置页

点击行为：

- **模式按钮（左侧）**：保持现有逻辑不变
- **设置按钮（右侧）**：
  - 若该模式有对应设置页：切换到对应 tab（即进入原来对应 tab 的内容区域）
  - 若无：`wx.showToast({ title: "功能还未开放", icon: "none" })`

### 3) 模式 → 设置页映射（确认版）

把原底部 Tab（图库/笔记/手绘）迁移为模式的“设置页入口”：

- **图片模式（mode=0）** → 进入 **图库设置页（tab=1）**
  - 复用现有「图片轮播 + 槽位上传」区域
- **笔记模式（mode=2）** → 进入 **笔记设置页（tab=2）**
  - 复用现有「笔记轮播 + 编辑 + 历史」区域
- **手绘模式（mode=4）** → 进入 **手绘设置页（tab=3）**
  - 复用现有「画板 + 颜色/粗细 + 清屏/你画我猜」区域
- 其余模式暂不提供设置页：
  - **时钟（mode=1）** → 未开放提示
  - **表情（mode=3）** → 未开放提示

备注：如果未来新增对应设置页，可更新映射即可。

## 实现落点（文件级）

- `WXCody/pages/console/console.wxml`
  - 改造 `modeGrid`：每个模式项增加右侧设置按钮（使用 `catchtap` 防止事件冒泡触发切模式）
  - 底部 `tabsBottom`：删减按钮到仅「模式/设置」
- `WXCody/pages/console/console.wxss`
  - 为“模式行 + 设置按钮”增加布局样式（行布局、右侧按钮尺寸等）
- `WXCody/pages/console/console.js`
  - 新增事件处理：`onOpenModeSettings`（或同名）
    - 根据 `mode` 做映射到 `tab`
    - 无映射则 toast
    - 有映射则复用现有 `onTab` 的切换逻辑（建议抽一个内部方法以避免复制）

## 错误处理 / 边界

- 未连接蓝牙时：
  - 进入设置页（tab 切换）允许发生（仅影响 UI），但该页里依赖 BLE 的刷新行为由既有代码控制（未连接会直接 return）
- 你画我猜进行中：
  - 现有 `onTab` 会阻止离开手绘页；模式设置按钮触发的 tab 切换应复用同一套拦截逻辑
- 事件冒泡：
  - 右侧设置按钮必须使用 `catchtap`，避免点击设置时触发 `onSetMode`

## 验收标准

- 底部只显示「模式」「设置」
- 模式页每个模式右侧有「设置」按钮
- 点模式按钮：仍能切换模式
- 点设置按钮：
  - 图片/笔记/手绘模式能进入各自设置页（实际为切换到 tab=1/2/3）
  - 时钟/表情提示“功能还未开放”
- 现有设置页（tab=4）仍可通过底部「设置」进入
